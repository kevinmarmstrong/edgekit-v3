browser-native agent runtime

Agent UX belongs inside the app, under the app's authority.

edgekit is a local-first sidecar runtime for enterprise web apps. Register existing functions as typed tools, hydrate identity and page state, run browser-native inference when available, and require approval before the agent changes important data.

First principles

The agent uses your app. The app stays in control.

Enterprise agent UX is not a detached chat panel. It is a governed interface over real app capabilities: identity, state, tools, approval, audit, and recovery.

01

Mount the sidecar

Add the edge-chat web component or use `createAgent()` when you need a custom interface.

02

Expose real actions

Wrap existing app functions as AI SDK tools with typed inputs and outputs.

03

Approve changes

Mark irreversible tools as guarded so the user approves before state changes.
