Open source agent workflows

Add an AI agent to your existing app. Don't rewrite the app.

Two lines of HTML. The agent runs in the user's browser, calls your existing APIs as tools, and falls back to the cloud only when it has to. Open source, MIT.

Quickstart (60 seconds) Star on GitHub Watch the 90s demo 
<edge-chat id="agent"></edge-chat>
import { chromeAI, webLLM } from '@kevinmarmstrong/edgekit'
agent.configure({ model: [chromeAI(), webLLM(), appCloudRoute] })
agent.registerTools({ searchProducts, addToCart })

First principles

The agent assists the workflow. Your app remains the authority.

edgekit is not asking you to move product logic into a chatbot. The agent gets a typed, governed view of what the app already knows how to do: identity, state, tools, approval, audit, and recovery.

01

Mount the agent UI

Add the edge-chat web component or use `createAgent()` when you need a custom interface.

02

Expose real actions

Wrap existing app functions as AI SDK tools with typed inputs and outputs.

03

Approve changes

Mark irreversible tools as guarded so the user approves before state changes.
